# 临时工排班论文故事线与可视化改进方案

生成时间：2026-05-01  
改进对象：大型展销会临时工招聘与排班优化问题  
参考材料角色：A066、A196 仅用于学习优秀数学建模论文的叙事组织、图表组织和结果展示方式，不迁移其无人机投弹模型内容。

## 1. 纠偏说明

本方案不再讨论无人机烟幕弹题目的建模与算法。A066、A196 是去年无人机投弹题的优秀论文，它们对当前临时工排班题的价值不在业务模型，而在论文写法：

1. 摘要如何快速交代每个问题的模型、算法和数值结果。
2. 正文如何用“问题分析-假设-符号-分问题模型-结果验证”形成稳定结构。
3. 图表如何服务论证，而不是只在结果部分堆叠。
4. 如何把复杂优化题写成递进故事线，而不是多个孤立算例。

因此，以下所有改进建议均面向当前临时工排班问题。

## 2. 从优秀论文抽取的可迁移范式

### 2.1 摘要范式

优秀论文摘要通常不是泛泛介绍，而是逐问题写清：问题对象、建模方法、求解算法、关键结果、验证方式。

迁移到临时工排班题，摘要应直接写出：

- 问题一：固定小组服务，列生成与整数主问题恢复，得到 417 人。
- 问题二：跨日可换组、日内单组，单日下界与最大流映射，得到 406 人并达到理论下界。
- 问题三：日内最多双组，异构任务模式、最大流与 ALNS 重构，得到 400 人并达到理论下界。
- 验证：覆盖缺口为 0，每名临时工工作 8 天，敏感性和稳定性实验支撑结果可靠性。

### 2.2 正文范式

优秀论文通常先统一问题语言，再分问题推进。临时工排班题建议采用：

1. 问题重述与约束梳理。
2. 问题分析：三问流动规则逐步放宽。
3. 模型假设与符号说明。
4. 基础数据处理与合法班型生成。
5. 问题一：固定小组服务的列生成模型。
6. 问题二：跨日换组的下界与最大流模型。
7. 问题三：日内双组服务的异构任务匹配模型。
8. 结果对比、敏感性分析与模型评价。

### 2.3 图表范式

优秀论文的图表有明确职责：

- 关系图负责交代问题递进关系。
- 流程图负责交代算法链路。
- 表格负责承载关键数值结果。
- 热力图、矩阵图、收敛图负责证明方案结构和算法过程。
- 敏感性图负责说明模型稳健性。

迁移到当前题目后，图表不应按“生成顺序”排列，而应按“论文证据链”排列。

## 4. 建议总故事线

建议全文主线改为：

**“流动规则逐步放宽下的分层人员调度优化：从固定小组列生成，到跨日换组最大流映射，再到日内双组异构任务重构。”**

这条主线能把三问自然串起来：

1. 问题一限制最强：员工全周期固定服务同一小组，因此需要把一个员工 10 天内的完整轨迹作为排班模式。
2. 问题二放宽跨日流动：员工不同日期可服务不同小组，因此单日覆盖和跨日人员身份绑定可以解耦。
3. 问题三进一步放宽日内流动：员工同一天最多服务两个小组，因此需要从单一班型扩展为异构任务模式。

论文要反复强调一个证据闭环：

**理论下界定标 → 可行排班构造 → 覆盖约束验证 → 工作天数验证 → 敏感性与稳定性验证。**

这样可以显著降低“像视频讲解复刻”的风险，因为论文的重心从“照搬求解步骤”转为“建立一套可解释、可验证的优化证据链”。
## 5. 摘要改进方案

建议摘要使用四段式。

第一段：说明题目本质。

> 大型展销会临时工招聘与排班问题可抽象为带时空覆盖、连续服务时长、工作天数和人员流动规则的人员调度优化问题。本文围绕三类流动约束逐步放宽的场景，建立分层排班模型，并用理论下界、可行构造和本地验证共同支撑求解结果。

第二段：写问题一。

> 针对问题一中临时工全周期固定服务同一小组的限制，本文将员工 10 天内的完整工作轨迹定义为排班列，建立集合覆盖形式的列生成模型。通过受限主问题、定价子问题和整数主问题恢复，得到 417 人的可行排班方案，并验证覆盖缺口为 0。

第三段：写问题二。

> 针对问题二中跨日可换组但日内只能服务一个小组的规则，本文先求解每日最少出勤人次，再由总工作日约束得到 406 人理论下界，并通过最大流模型完成员工-日期映射，构造出 406 人可行方案，实现下界达到。

第四段：写问题三与验证。

> 针对问题三中日内最多服务两个小组的规则，本文枚举单组 8 小时与双组 4+4 小时异构任务模式，结合单日覆盖优化、最大流初始化和 ALNS 重构，得到 400 人方案。结果显示覆盖缺口为 0，每名员工恰好工作 8 天；敏感性和多随机种子实验进一步说明模型具有较好的稳健性。

摘要最后给关键词：临时工排班；列生成；最大流；异构任务匹配；ALNS；敏感性分析。

## 6. 章节级改进方案

### 6.1 问题重述与问题分析

应新增“三问约束递进表”，放在问题分析开头。

| 维度 | 问题一 | 问题二 | 问题三 |
|---|---|---|---|
| 跨日是否可换组 | 否 | 是 | 是 |
| 日内是否可换组 | 否 | 否 | 是，最多两个小组 |
| 日内服务结构 | 同一小组 8 小时 | 同一小组 8 小时 | 单组 8 小时或双组 4+4 小时 |
| 主要建模难点 | 全周期固定小组导致模式组合大 | 单日覆盖与跨日身份绑定 | 异构任务模式与休息约束耦合 |
| 推荐求解线索 | 列生成 | 单日下界 + 最大流 | 异构模式 + 最大流 + ALNS |

配套正文应说明：三问不是三个独立问题，而是同一调度问题在人员流动限制逐步放宽下的三个层级。

### 6.2 模型假设与符号说明

建议统一符号，不要每问重新造一套。

| 符号 | 含义 |
|---|---|
| \(D\) | 日期集合，\(D=\{1,\dots,10\}\) |
| \(G\) | 小组集合，\(G=\{1,\dots,10\}\) |
| \(T\) | 小时集合 |
| \(r_{dgt}\) | 第 \(d\) 天第 \(g\) 组第 \(t\) 小时需求人数 |
| \(S\) | 合法单组 8 小时班型集合 |
| \(B_d\) | 问题三第 \(d\) 天异构任务模式集合 |
| \(x_p\) | 是否选择完整排班列 \(p\) |
| \(m_d\) | 第 \(d\) 天最少出勤人次 |
| \(N\) | 招聘临时工人数 |
模型假设建议保持克制：

1. 附件给出的小时需求为必须满足的刚性覆盖需求。
2. 每名临时工全周期恰好工作 8 天。
3. 每个工作日由两个连续 4 小时工作段构成。
4. 问题三跨组服务需满足题目规定的休息间隔。
5. 只优化招聘人数，超额覆盖作为辅助评价指标，不作为主目标。

### 6.3 基础数据处理章节

优秀论文通常会在模型前解释数据如何进入模型。当前论文建议补一节“需求张量与合法班型生成”。

应写清三件事：

1. 将附件需求整理为三维张量 \(r_{dgt}\)。
2. 枚举每天合法 8 小时服务班型。
3. 问题三进一步枚举双组 4+4 小时异构模式，并过滤不满足休息间隔的模式。

配图建议：使用 `fig01_model_framework` 前置展示完整求解链路。

### 6.4 问题一章节

问题一建议标题：**固定小组服务约束下的完整模式列生成模型**。

章节结构：

1. 固定小组约束分析。
2. 完整排班列定义。
3. 受限主问题 RMP。
4. 定价子问题与列生成过程。
5. 整数主问题恢复与结果验证。

应插入图表：

- `fig03_q1_column_generation`：放在算法结果部分。
- 问题一列生成结果表：小组、最终列池规模、新增列数、LP 目标、整数人数、人工松弛。

正文重点：

- 不要只说“用列生成求解”，要说明为什么列生成适合问题一：完整枚举所有 10 天轨迹规模大，但每个轨迹天然满足固定小组和工作 8 天约束。
- 如果没有完整 solver gap，不要轻率写“严格全局最优”；可以写“在生成列池上整数恢复，并通过人工松弛为 0 验证可行性”。若已有下界证明，则用“下界与可行构造相等”表述。

### 6.5 问题二章节

问题二建议标题：**跨日换组场景下的下界定标与最大流映射模型**。

章节结构：

1. 单日覆盖与跨日人员身份解耦。
2. 每日最少出勤人次计算。
3. 理论人数下界推导。
4. 最大流网络构造。
5. 406 人方案与下界达到证明。

应插入图表：

- `fig04_daily_attendance`：展示每日最少出勤与最大流安排人数。
- 新增最大流网络示意图：源点、员工节点、日期节点、汇点。
- 问题二每日出勤表：当前已有 3247 总出勤下界和 3248 实际工作日。

正文重点：

- 强调 406 不是“程序搜索出来的经验值”，而是 \(\lceil3247/8\rceil\) 的理论下界。
- 最大流的作用是证明这个下界可以被具体员工工作日安排实现。
- 解释多出的 1 个出勤人次来自 3247 不能被 8 整除，不影响人数最优性。

### 6.6 问题三章节

问题三建议标题：**日内双组服务场景下的异构任务匹配与 ALNS 重构模型**。

章节结构：

4. 理论下界与最大流初始化。

- `q3_operator_ablation.csv` 对应消融表。
- `q3_seed_stability.csv` 对应多随机种子稳定性表。

正文重点：

- ALNS 的定位要准确：它是异构任务匹配和排班重构工具，不是 400 人最优性的唯一证明。

## 9. 结果展示方式优化

### 9.1 先给总结果，再给分问题细节

优秀论文常常先让评委看到结果，再解释为什么可信。建议在三问模型前或模型后立即放总结果表：

- 问题一 417 人。
- 问题二 406 人。
- 问题三 400 人。
- 相比问题一，问题二节约 11 人，问题三累计节约 17 人。

然后再分问题解释模型。

### 9.2 每个问题形成固定四件套

每个问题小节都采用同一结构：

1. 约束变化。
2. 模型构造。
3. 求解方法。
4. 结果与验证。

这样读者不需要重新适应章节逻辑。

### 9.3 图注要写结论，不只写名称

例如不要只写“图4 每日出勤结构对比”，而应写：

> 图4 显示问题二总最少出勤为 3247，问题三因允许日内双组服务降至 3198，说明流动规则放宽直接降低了招聘人数下界。

图注本身要承担解释任务。
## 10. 算法增强策略：不夸张版本

用户提出“增强策略不能太夸张”，当前临时工排班题建议只做以下克制增强。

### 10.1 问题一补下界解释

当前列生成结果已有 417，但论文里最容易被质疑的是“为什么 417 是最少”。建议补充：

- 每组 LP 松弛下界。
- 每组总出勤人次下界。
- 每组整数恢复人数。
- 若可行解人数等于组合下界，则明确写“下界达到”。
- 若仍有 gap，则不要宣称严格最优，只写“在列生成列池上的最优整数恢复结果”。

这是必要增强，不是算法堆叠。

### 10.2 问题三目标解释增强

ALNS 当前已经达到可行 400 人，继续说它“优化人数”会显得夸张。建议把 ALNS 的优化目标改写为：

- 保持覆盖缺口为 0。
- 保持每人工作 8 天。
- 降低覆盖冗余平方或最大冗余。
- 改善每日出勤均衡性。

这样 ALNS 的存在更合理：它不是为了证明人数最优，而是为了让可行排班结构更好。

### 10.3 消融实验增强

已有 `q3_operator_ablation.csv`，建议正文放一个小表即可。重点说明：不同算子组合都能保持覆盖缺口为 0，增强目标可降低冗余平方或改善结构指标。

不建议把消融写成深度学习式大实验，只要支撑 ALNS 设计有效即可。
