# 临时工排班论文组织初稿（详细版）

生成时间：2026-05-01 15:30:00

## 1. 这版初稿解决什么问题

这份初稿不是重新发明模型，而是把当前项目已经完成的三问求解、最优性证据、图表素材和往年优秀论文的组织方式整合成一套可直接落稿的论文蓝图。写作目标有三个：

1. 让评委在前 2 页内看清题目对象、三问递进和最终结果。
2. 让正文每一章都围绕“下界 / 构造 / 验证”的证据链展开，而不是堆算法名词。
3. 让现有 9 张图和现成结果表都各有职责，形成一条完整的论证路径。

## 2. 论文总标题候选

建议从以下三个标题中选一个：

1. 大型展销会临时工招聘与排班优化的分层建模与求解
2. 流动规则逐步放宽下的临时工招聘与排班优化研究
3. 基于列生成、网络流与异构任务重构的临时工排班优化模型

更推荐第 2 个标题，因为它天然对应三问的递进关系，也更容易带出全文故事线。

## 3. 全文核心故事线

全文不要写成“问题一用了列生成，问题二用了最大流，问题三用了 ALNS”的算法流水账，而要写成下面这条业务主线：

> 本文研究大型展销会临时工招聘与排班优化问题。随着人员流动规则逐步放宽，可行域不断扩大，招聘规模逐步下降。为此，本文围绕固定小组、跨日换组、日内双组三类场景，采用合法班型池驱动的覆盖优化方法，分别建立列生成精确模型、理论下界与最大流映射模型、异构任务匹配与 ALNS 重构模型，并通过理论下界、可行排班构造、本地验证和鲁棒性评价形成证据闭环。

这条主线要在四个地方反复出现：

1. 摘要第一段。
2. 引言末尾的研究框架概述。
3. 问题分析章节的关系图说明。
4. 结论部分的总括段。

## 4. 推荐摘要结构

摘要建议用四段式，每段各有职责。

### 4.1 第一段：问题本质

说明这是一个带时空覆盖、连续服务时长、工作天数和人员流动规则的人员调度优化问题，三问只是流动约束不同，不是三个无关问题。

### 4.2 第二段：问题一

要直接写出：

- 固定小组服务；
- 列生成 + 整数主问题恢复；
- 得到 417 人；
- 417 不是经验结果，而是有精确恢复与可行性验证支撑。

### 4.3 第三段：问题二与问题三

按“下界 → 构造 → 结果”顺序写：

- 问题二：每日最少出勤人数 + 总工作日下界 + 最大流映射，得到 406 人并达到理论下界；
- 问题三：异构任务模式 + 理论下界 + 最大流初始化 + ALNS 重构，得到 400 人并达到理论下界。

### 4.4 第四段：验证与价值

交代：

- 覆盖缺口为 0；
- 每名临时工工作 8 天；
- `417/406/400` 是理想条件下的理论极限配置，不等价于现实中唯一稳妥的执行人数；
- 灵敏度分析说明原方案在需求放大和随机缺勤下会出现缺口，因此需把容量缓冲和待命机制放到模型推广中讨论。

### 第五章 问题二：跨日换组下的下界与最大流映射模型

这一章要突出“解耦”思想。

#### 5.1 问题二场景分析

## 12. 实际写作顺序建议

正式落稿时，不要从第一章开始硬写。更高效的顺序是：

1. 先写“三问结果总表”和图表题注；
2. 再写第四章、第五章、第六章的模型与结果；
3. 接着写第三章的问题分析与总体框架；
4. 再补第二章的假设与符号；
5. 最后写第一章摘要、引言和第九章结论。

这样可以确保前文不是空转，而是被后文真实结果反向约束。
